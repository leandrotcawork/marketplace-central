# Marketplaces Page Redesign — Design Spec

**Date:** 2026-04-09
**Status:** Approved
**Scope:** `packages/feature-marketplaces` — `MarketplaceSettingsPage.tsx` and supporting UI components

---

## 1. Problem Statement

The current Marketplaces page has two fundamental problems:

1. **Information architecture:** Accounts and Policies are two disconnected sections stacked vertically. The relationship between them is invisible — a user has no clear mental model that a policy belongs to an account.
2. **No design system:** Raw Tailwind with no hierarchy, no visual identity, no consistent card pattern, no loading/empty/error states beyond minimal implementations.

The redesign solves both: it unifies Account + Policy into one object per card, and applies the project design system (glassmorphism, Plus Jakarta Sans, `#2563EB` primary, `#F97316` CTA).

## 3. Data Model (unchanged)

The page continues to use the existing API via `sdk-runtime`. No new endpoints required.

**Account fields:** `account_id`, `display_name`, `channel_code`, `marketplace_code`, `connection_mode`, `credentials_json`, `status`

**Policy fields:** `policy_id`, `account_id`, `commission_percent`, `fixed_fee_amount`, `default_shipping`, `min_margin_percent`, `sla_question_minutes`, `sla_dispatch_hours`

**ID generation:** In the redesigned UX, `account_id` and `policy_id` are auto-generated by the frontend (e.g. `acc-<uuid4-short>`, `pol-<uuid4-short>`) and never shown as editable fields. Users only fill in human-facing fields (display name, credentials, fees). This eliminates a current pain point where users must invent and type IDs manually.

**Definitions:** loaded from `listMarketplaceDefinitions()` — provides `display_name`, `credential_schema[]` per marketplace type.

The redesign treats Account + Policy as one unified object per card. The `account_id` is the link between them.

```
┌─────────────────────────────────────────────────────────┐
│  Marketplaces                        [+ Connect]         │  Page header
│  Manage your channels and pricing policies               │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  [Card]   [Card]   [Card]                                │  Card grid
│  [Card]   [Card]   [+ Add card]                          │  3→2→1 cols
│                                                          │
└─────────────────────────────────────────────────────────┘
                                          ┌───────────────┐
                                          │  Detail Panel │  400px, fixed right
                                          │               │  grid shifts left
                                          └───────────────┘
```

- **Page header:** `<h1>Marketplaces</h1>` + subtitle + `[+ Connect Marketplace]` button (orange CTA, right-aligned)
- **Card grid:** CSS Grid, `repeat(auto-fill, minmax(280px, 1fr))`, gap `1.5rem`. Works from 1 to 20+ accounts.
- **Detail panel:** 400px fixed width, slides in from the right. The grid container gets `padding-right: 400px` when panel is open — cards reflow, nothing is hidden.
- **Empty state:** When no accounts exist, replace the grid with a centered empty state (icon + headline + CTA button). See Section 7.

## 5. Account Card

### Anatomy

```
┌─────────────────────────────────┐
│  [Marketplace icon]  ● active   │  Row 1: icon (32px) + status badge
│                                  │
│  My VTEX Store                   │  Display name — font-semibold, text-slate-900
│  acc-001 · vtex                  │  ID + channel — text-xs, text-slate-400, mono
│                                  │
│  ────────────────────────────   │  Divider
│                                  │
│  Commission      Min Margin      │  Policy snapshot labels — text-xs, text-slate-500
│  16.0%           10.0%           │  Policy snapshot values — font-bold, text-slate-900
│                                  │
└─────────────────────────────────┘
```

### Card Styles

| State | Style |
|-------|-------|
| Default | `bg-white`, `border border-slate-100`, `shadow-sm`, `rounded-2xl`, `p-5` |
| Hover | Shadow increases to `shadow-md`, border shifts to `border-blue-100` (150ms) |
| Selected (panel open) | `border-2 border-blue-500`, `bg-blue-50/30` |
| Inactive account | Same structure, status dot is `text-slate-400`, policy values in slate |

### Status Badge
- Active: `bg-emerald-100 text-emerald-700` pill + `●` dot
- Inactive / other: `bg-slate-100 text-slate-500` pill

### Policy Snapshot
- Shows `commission_percent * 100` as `16.0%` and `min_margin_percent * 100` as `10.0%`
- If no policy configured: show `—` dashes + small `"Configure policy →"` link in `text-blue-600`

### Marketplace Icon
- 32×32px rounded square per marketplace, using brand color as background with white initial letter (fallback when no logo asset)
- Color mapping: VTEX → `#FF3366`, Mercado Livre → `#FFE600` (dark text), Magalu → `#0086FF`, Shopee → `#EE4D2D`, Americanas → `#E30613`

## 8. Detail Panel

### Header

```
┌─────────────────────────────────────┐
│  [MP icon]  My VTEX Store     [✕]   │
│             acc-001 · active         │
├─────────────────────────────────────┤
```

- Marketplace icon (32px) + display name (`font-semibold text-slate-900`) + close button (Lucide `X`, `text-slate-400`)
- Subtitle: account ID + status badge inline
- Panel slides in with `transform: translateX(0)` from `translateX(100%)` (200ms ease-out)

### CONNECTION Section

Collapsible (open by default). Header: `text-xs font-semibold text-slate-500 uppercase tracking-wide` + chevron toggle.

**Create mode:** all fields editable
**Edit mode:** Marketplace type is read-only (shown as a pill, not an input). All other fields editable.

Fields:
- Display Name (text)
- Account ID (text, read-only after creation — shown as `font-mono text-slate-600`)
- Connection Mode (text)
- Marketplace (dropdown from definitions — create mode only)
- Credentials: dynamic fields from `credential_schema[]`. Secret fields (`field.secret === true`) use `type="password"` with a Lucide `Eye`/`EyeOff` toggle button

### PRICING POLICY Section

Collapsible (open by default). Header same style as CONNECTION.

Fields displayed in a 2-column grid:
- Commission % (number, placeholder `0.16`)
- Fixed Fee R$ (number, placeholder `5.00`)
- Default Shipping R$ (number, placeholder `10.00`)
- Min Margin % (number, placeholder `0.10`)
- SLA Question (number, suffix "min")
- SLA Dispatch (number, suffix "h")

**Note on commission_percent:** stored and sent as decimal (0.16 = 16%). The label must read `Commission (0.16 = 16%)` to avoid user confusion. Display in the card multiplies by 100 for readability.

### Panel Footer

```
├─────────────────────────────────────┤
│  [Disconnect]          [Save Changes]│
└─────────────────────────────────────┘
```

- `[Disconnect]`: `text-sm text-red-600 hover:text-red-800`, left-aligned. Shows a confirmation step inline (replaces button text with "Are you sure? [Yes, disconnect]") — no modal. Disconnecting removes the account AND its associated policy from the UI. Both are deleted from the backend. The card disappears from the grid after success.
- `[Save Changes]`: orange CTA button, right-aligned. Disabled until any field changes. Shows loading spinner on submit.
- **Create mode footer:** `[Cancel]` (secondary) + `[Connect]` (orange CTA)

### Panel States

| State | Behavior |
|-------|----------|
| Loading panel data | Skeleton rows in both sections |
| Save success | Brief `✓ Saved` green flash in footer, button returns to disabled |
| Save error | Red error message appears above footer |
| No policy yet | PRICING POLICY section shows empty fields with a note: "No policy configured yet — fill in to activate." |

## 9. Loading & Error States

### Page Loading
- Header renders immediately (title + CTA button visible)
- Grid shows 3 skeleton cards (animated pulse, same dimensions as real cards)

### Page Error
- Red banner below header: `Failed to load marketplaces. [Retry]`
- Grid area empty

### Definitions Loading (within create panel)
- Marketplace dropdown shows `Loading marketplaces…` disabled option

## 11. Accessibility

- All form inputs have `<label>` with `htmlFor` — no placeholder-only labels
- Panel has `role="dialog"` and `aria-label="Account settings"`
- Focus trapped inside panel when open; returns to triggering card on close
- Status badges have `aria-label` with full text (not just color)
- `prefers-reduced-motion`: disable slide-in transition, use instant show/hide
